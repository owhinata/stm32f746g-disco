# STM32F746G-DISCO — bare-metal LED blink

OS-less LED blink sample for the STM32F746G-DISCO (STM32F746NGH6), built with
the **official STMicroelectronics HAL**. The HAL/CMSIS sources and the ARM GNU
toolchain are both fetched automatically — nothing to install by hand except
`git`, `make`, `curl`, and (for flashing) `st-flash`.

## Chip configuration

| Item    | Setting                                            |
|---------|----------------------------------------------------|
| Core    | Cortex-M7, hard-float (`fpv5-sp-d16`) — **FPU on** |
| I-Cache | **enabled** (`SCB_EnableICache`)                   |
| D-Cache | **enabled** (`SCB_EnableDCache`)                   |
| SYSCLK  | **200 MHz** from 25 MHz HSE (PLL M=25, N=400, P=2) |
| Power   | VOS scale 1 + over-drive, 6 flash wait states      |
| LED     | LD1 (green) on **PI1**, blinking at 1 Hz           |

## Layout

```
include/   main.h, stm32f7xx_it.h
src/       main.c, stm32f7xx_it.c
ldscript/  STM32F746NGHx_FLASH.ld
lib/       git submodules:
             stm32f7xx_hal_driver   (ST official HAL)
             cmsis_device_f7        (startup, system_*, device headers)
             cmsis_core             (CMSIS Core, cm7 branch)
Makefile   downloads the toolchain + builds
```

`stm32f7xx_hal_conf.h` is generated by the Makefile from the upstream HAL
template (its default `HSE_VALUE` of 25 MHz already matches the board crystal).

## Build

```bash
git clone --recurse-submodules <this repo>
cd stm32f746g-disco
make            # first run downloads the ARM toolchain (~150 MB) into ./tools
```

Output: `build/blink.elf`, `build/blink.hex`, `build/blink.bin`.

If you cloned without `--recurse-submodules`, the Makefile runs
`git submodule update --init` for you on the first build.

## Flash / run

```bash
make flash      # st-flash write build/blink.bin 0x08000000, then reset
make reset      # just reset the target
```

The green LD1 LED blinks at 1 Hz.

## Notes

- No software bootloader: the application is linked at `0x08000000` and uses the
  full 1 MB flash. Field updates can use the STM32 ROM bootloader (BOOT0 high →
  USB DFU / UART) without changing this image.
- `make distclean` removes the downloaded toolchain as well as build artifacts.
