# STM32F746G-DISCO — bare-metal apps (blink / CoreMark)

OS-less samples for the STM32F746G-DISCO (STM32F746NGH6), built with the
**official STMicroelectronics HAL**. The HAL/CMSIS sources and the ARM GNU
toolchain are both fetched automatically — nothing to install by hand except
`git`, `make`, `curl`, and (for flashing) `st-flash`.

Two apps share the same board bring-up (`src/bsp.c`):

| App        | What it does                                                       |
|------------|-------------------------------------------------------------------|
| `blink`    | Blinks LD1 (PI1) at 1 Hz, prints `hello world` + a tick over VCP   |
| `coremark` | Runs EEMBC CoreMark and prints the score over the VCP             |

## Chip configuration

| Item    | Setting                                            |
|---------|----------------------------------------------------|
| Core    | Cortex-M7, hard-float (`fpv5-sp-d16`) — **FPU on** |
| I-Cache | **enabled** (`SCB_EnableICache`)                   |
| D-Cache | **enabled** (`SCB_EnableDCache`)                   |
| SYSCLK  | **216 MHz** from 25 MHz HSE (PLL M=25, N=432, P=2) |
| Power   | VOS scale 1 + over-drive, 7 flash wait states      |
| VCP     | USART1, TX=PA9 / RX=PB7, 115200 8N1 (ST-Link VCP)  |
| LED     | LD1 (green) on **PI1**                              |

## Layout

```
include/        main.h, bsp.h, stm32f7xx_it.h
src/            bsp.c (clock/cache/UART/printf), main.c (blink), stm32f7xx_it.c
port/coremark/  core_portme.{c,h}   CoreMark port (SysTick timing, printf output)
ldscript/       STM32F746NGHx_FLASH.ld
lib/            git submodules:
                  stm32f7xx_hal_driver   (ST official HAL)
                  cmsis_device_f7        (startup, system_*, device headers)
                  cmsis_core             (CMSIS Core, cm7 branch)
                  coremark               (EEMBC CoreMark)
Makefile        downloads the toolchain + builds (APP=blink|coremark)
```

`stm32f7xx_hal_conf.h` is generated by the Makefile from the upstream HAL
template (its default `HSE_VALUE` of 25 MHz already matches the board crystal).

## Build

```bash
git clone --recurse-submodules <this repo>
cd stm32f746g-disco
make                 # APP=blink (default); first run downloads the toolchain
make APP=coremark    # build CoreMark instead
```

Artifacts land in `build/<app>/<app>.{elf,hex,bin}`. If you cloned without
`--recurse-submodules`, the Makefile runs `git submodule update --init` for you.

## Flash / run

```bash
make flash                 # blink
make APP=coremark flash    # CoreMark
make reset                 # just reset the target
```

Watch the VCP (`/dev/ttyACM0`) with any serial terminal at 115200 8N1, e.g.:

```bash
picocom -b 115200 --imap lfcrlf /dev/ttyACM0
```

## CoreMark result

Measured on this board (216 MHz, GCC 13.3, `-O3`, code in flash with I/D-cache,
performance run, auto-calibrated to ~11.8 s / 11000 iterations):

```
2K performance run parameters for coremark.
CoreMark Size    : 666
Total time (secs): 11.843000
Iterations/Sec   : 928.818711
Iterations       : 11000
Compiler version : GCC13.3.1 20240614
Compiler flags   : -O3 -mcpu=cortex-m7 -mfpu=fpv5-sp-d16 -mfloat-abi=hard
Memory location  : STACK
Correct operation validated. See README.md for run and reporting rules.
CoreMark 1.0 : 928.818711 / GCC13.3.1 20240614 -O3 ... / STACK
```

**CoreMark ≈ 928.8**, i.e. **≈ 4.30 CoreMark/MHz**. The port times the run with
the 1 ms SysTick (`HAL_GetTick`) and prints through the VCP `printf`; CoreMark
self-calibrates the iteration count (`ITERATIONS=0`) to run 10–100 s. Running
code from flash (7 wait states) caps the per-MHz figure; placing the hot code in
ITCM/RAM would push it higher.

## Notes

- No software bootloader: each app is linked at `0x08000000` and uses the full
  1 MB flash. Field updates can use the STM32 ROM bootloader (BOOT0 high → USB
  DFU / UART) without changing the image.
- `make distclean` removes the downloaded toolchain as well as build artifacts.
- The VCP `printf` float support (`%f` in CoreMark) is enabled by linking with
  `-u _printf_float` (nano newlib), set automatically for `APP=coremark`.
